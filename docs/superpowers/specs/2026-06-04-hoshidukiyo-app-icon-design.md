# Hoshidukiyo（星月夜）アプリアイコン設計書

- 日付: 2026-06-04
- ステータス: 設計合意済み（実装計画はこの後 `writing-plans` で作成）
- 作業リポジトリ: `hoshidukiyo`（アプリ名 Hoshidukiyo / 星月夜）
- 対象: macOS アプリ `Hoshidukiyo` の AppIcon

## 1. 概要

Hoshidukiyo の macOS アプリアイコンを新規に作成する。アプリは夜フクロウ（Yorufukurou）に倣った Bluesky クライアントであり、アプリ名「星月夜（＝月の出た星空の夜）」にちなみ、夜にたたずむ**ミミズク**をモチーフにする。アイコンの「絵」は CC0（パブリックドメイン相当）のミミズク SVG 素材を採用し、それをアプリの配色（ダーク地＋ブルーアクセント）と立体感のある表現に仕立てて、macOS 用の AppIcon 一式（16〜1024px、@2x 含む）として書き出す。

絵柄は AI 画像生成や手描きではなく、ライセンスの明確な既存 CC0 素材を起点とする。背景・配色・陰影など素材以外の構成要素は自前で生成するため、著作権・商標上の懸念はない。

## 2. ゴール / 非ゴール

### ゴール（本設計が対象とする範囲）
- CC0 のミミズク SVG を採用素材として確定し、リポジトリに出典とともに保管する
- アプリの世界観（ダーク＋ブルー、星月夜＝夜）に合わせた 1024px マスターアイコンを生成する
- macOS の `AppIcon.appiconset`（必要全サイズ ＋ `Contents.json`）を生成し、Xcode プロジェクトに組み込んでアプリのアイコンとして反映する
- 元 SVG と生成スクリプトをリポジトリに置き、アイコンを再現・再生成できる状態にする

### 非ゴール
- iOS / その他プラットフォーム用アイコン（コアと同様、別途必要になった時点で対応）
- メニューバー用テンプレートアイコン（モノクロ・自動着色対応）の作成。本 spec はアプリ本体の AppIcon のみ
- マーケティング用バナーや App Store 用素材

## 3. 意思決定ログ（ブレインストーミングでの確定事項）

| 項目 | 決定 |
|---|---|
| モチーフ | ミミズク（フクロウではない） |
| 素材の作り方 | CC0 素材から探す。SVG 形式が望ましい |
| 採用素材 | The Noun Project のミミズクアイコン（CC0）。Wikimedia Commons 経由で取得 |
| フクロウ/ミミズクの区別 | 羽角（うかく＝頭頂の耳状の飾り羽）の有無が違い。羽角ありがミミズク。採用素材は羽角を持ちミミズクの条件を満たす |
| 構図の方向性 | B 案（アプリ背景色のダーク地＋ブルーのミミズク）をベースに採用 |
| 立体感 | B3（ミミズク本体に上から光が当たるグラデ＋斜めグラデ背景＋ドロップシャドウ）。「のっぺり」を本体の陰影で解消 |
| 形状 | macOS 流の角丸スクエア（squircle 近似）。余白・角丸・影を PNG に焼き込む |

### 3.1 フクロウとミミズクの違い（調査結果）

両者の最大の違いは**羽角（うかく）**の有無である。羽角は頭頂部の両側にある「耳のように見える飾り羽」で、実際の耳でも聴覚器官でもない（木の葉への擬態に役立つという説がある）。羽角がないものがフクロウ、あるものがミミズクと呼ばれる。コノハズクは最小種のミミズク、ワシミミズク（英名 Eurasian Eagle Owl）は最大種。名前の例外として、シマフクロウは「フクロウ」だが羽角があり、アオバズクは「ズク」だが羽角がない。本アイコンの採用素材は頭頂に 2 つの羽角を持つため、ミミズクとして成立する。

出典:
- https://kururu-owl.com/mimizuku/
- https://akiba2960.com/jp/mimizuku/
- https://cuteanimalmuseum.com/429/

## 4. 採用素材とライセンス

- 素材名: Owl (20398) - The Noun Project
- 取得元: Wikimedia Commons `File:Owl (20398) - The Noun Project.svg`
- ライセンス: **CC0 1.0（パブリックドメイン）**
- 形式: SVG（`viewBox="0 0 100 100"`、単色パス。フクロウ本体 ＋ 2 つの目の円 ＋ 頭頂の羽角）
- 帰属表示: CC0 のため法的には不要。ただしリポジトリに出典・ライセンスを記録する（`design/app-icon/SOURCE.md` 等）
- 加工メモ: Adobe Illustrator 由来の `<switch>/<foreignObject>` ラッパと、未定義名前空間属性（`i:` 等）を除去してから利用する（標準 SVG レンダラ `rsvg-convert` で解釈できるようにするため）

## 5. ビジュアル仕様（B3）

- キャンバス: 1024 × 1024px（マスター）
- 角丸スクエア本体: キャンバス内に上下左右 約100px の余白を取り、824 × 824px の角丸矩形（角丸半径 約185px）を中央配置
- 背景グラデ（角丸本体内）: 斜め（左上→右下）`#222b40 → #171a22 → #0f1014`。アプリの `Theme.background`（おおよそ `#1c1c21`）系のダーク
- ミミズク本体グラデ: 上明→下濃 `#bcd0fb → #6f99f0 → #4a72d6`。アプリの `Theme.accent`（おおよそ `#5c8fed`）系ブルー。上からの光で量感を出す
- 影:
  - 角丸本体に外側ドロップシャドウ（`dy≈18`, `blur≈22`, 黒 50%）
  - ミミズク本体に柔らかいドロップシャドウ（`dy≈10`, `blur≈14`, 黒 45%）で地から浮かせる
- ミミズク配置: 角丸本体内に中央配置（バウンディングボックス中心を本体中心に合わせる）。本体に対して概ね 6 割強のサイズ
- macOS 特有事項: macOS は iOS と異なりアプリアイコンを自動でマスクしない。したがって角丸・余白・影はすべて PNG に焼き込む

### 微調整の余地
ミミズク素材は単一の弧を描く非対称な形状で、左右対称ではない。現状はこの素材本来の形を活かして中央配置するが、最終確認で「もう少し中央寄せ／拡大／左右対称化」の要望があれば調整する。

## 6. 生成する AppIcon 一式（macOS）

`AppIcon.appiconset` に以下を用意する。マスター SVG から各ピクセルサイズへ `rsvg-convert` で書き出す（ベクタから直接書き出すことで各サイズが鮮明になる）。

| 用途サイズ | スケール | 実ピクセル |
|---|---|---|
| 16pt | 1x / 2x | 16 / 32 |
| 32pt | 1x / 2x | 32 / 64 |
| 128pt | 1x / 2x | 128 / 256 |
| 256pt | 1x / 2x | 256 / 512 |
| 512pt | 1x / 2x | 512 / 1024 |

- 必要な実ピクセル: 16, 32, 64, 128, 256, 512, 1024（32 / 256 / 512 は複数エントリで共用）
- `Contents.json` は macOS（`"idiom": "mac"`）の上記 10 エントリで構成する
- 小サイズ（16 / 32px）でも目とシルエットが判別できることを確認済み

## 7. リポジトリへの配置と組み込み

- ソース・オブ・トゥルース: `design/app-icon/` に以下を置く
  - 採用素材の生 SVG（クリーン化前 / 後）
  - 出典・ライセンス記録（`SOURCE.md`）
  - 生成スクリプト（マスター SVG 合成 ＋ 全サイズ書き出し）
- 生成物: `app/Hoshidukiyo/Assets.xcassets/AppIcon.appiconset/`（各サイズ PNG ＋ `Contents.json`）
  - 注: 現状 `app/Hoshidukiyo` 配下に `Assets.xcassets` は未存在。新規作成する
- Xcode プロジェクト（XcodeGen `project.yml`）:
  - `Assets.xcassets` は既存の `sources: app/Hoshidukiyo` 配下に入るため自動的にビルド対象になる
  - ビルド設定に `ASSETCATALOG_COMPILER_APPICON_NAME: AppIcon` を追加し、アプリが AppIcon を参照するようにする
  - `xcodegen generate` で `.xcodeproj` を再生成して反映

## 8. 依存ツール

- `rsvg-convert`（SVG → PNG ラスタライズ。導入済みを確認済み）
- 補助的に ImageMagick（`magick`。確認用コンタクトシート等。必須ではない）
- `xcodegen`（`project.yml` からの `.xcodeproj` 再生成）

## 9. 完了条件

1. `design/app-icon/` に元 SVG・出典・生成スクリプトが揃い、スクリプト実行で AppIcon 一式が再生成できる
2. `app/Hoshidukiyo/Assets.xcassets/AppIcon.appiconset/` に全サイズ PNG と妥当な `Contents.json` が存在する
3. `project.yml` に `ASSETCATALOG_COMPILER_APPICON_NAME` が設定され、`xcodegen generate` 後にビルドが通る
4. ビルドしたアプリの Dock / Finder 表示で、立体感のあるミミズクアイコンが表示される
5. 16 / 32px の小サイズでもアイコンが破綻しない
